# How to Get Actual Environment Variable Values

## Frontend Environment Variables (.env)

### 1. REACT_APP_CLOUD_NAME_CLOUDINARY
Steps to get your Cloudinary cloud name:
1. Go to https://cloudinary.com/users/register_free
2. Sign up for a free account
3. After registration, you'll be taken to your dashboard
4. Look for "Account Details" section
5. Your cloud name will be displayed there (e.g., "ecommerce-shop-12345")

### 2. REACT_APP_STRIPE_PUBLIC_KEY
Steps to get your Stripe public key:
1. Go to https://dashboard.stripe.com/register
2. Sign up for a Stripe account
3. After registration, go to Developers > API Keys
4. Copy the "Publishable key" (starts with pk_test_)

### 3. REACT_APP_BACKEND_URL
This will be your deployed backend URL:
- During local development: http://localhost:8080
- After deployment: https://your-backend-app.onrender.com

## Backend Environment Variables (.env)

### 1. MONGODB_URI
Steps to get MongoDB connection string:
1. Go to https://www.mongodb.com/cloud/atlas/register
2. Sign up for MongoDB Atlas (free tier available)
3. Create a cluster
4. Create a database user
5. Add your IP address to the whitelist (or allow 0.0.0.0/0 for development)
6. Click "Connect" > "Connect your application"
7. Copy the connection string and replace <password> with your actual password

### 2. FRONTEND_URL
This will be your deployed frontend URL:
- During local development: http://localhost:3000
- After deployment: https://your-frontend-app.vercel.app

### 3. TOKEN_SECRET_KEY
Generate a secure random string:
- Use a password generator or create a long random string
- Example: "my-super-secret-jwt-key-2025!"

### 4. STRIPE_SECRET_KEY
Steps to get your Stripe secret key:
1. Go to your Stripe dashboard
2. Go to Developers > API Keys
3. Copy the "Secret key" (starts with sk_test_)

### 5. STRIPE_ENDPOINT_WEBHOOK_SECRET_KEY
Steps to get your webhook secret:
1. Go to your Stripe dashboard
2. Go to Developers > Webhooks
3. Add an endpoint or use an existing one
4. Reveal the signing secret

## How to Configure in Vercel (Frontend)

1. Go to your Vercel dashboard
2. Select your project
3. Click "Settings" > "Environment Variables"
4. Add each variable with its actual value:
   - Key: REACT_APP_CLOUD_NAME_CLOUDINARY
   - Value: your_actual_cloud_name
   - Environments: Select all
5. Repeat for other variables
6. Redeploy your application

## How to Configure in Render (Backend)

1. Go to your Render dashboard
2. Select your service
3. Click "Environment" in the sidebar
4. Add each variable with its actual value
5. The service will automatically redeploy
